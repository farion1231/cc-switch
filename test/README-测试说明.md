## 如何在 Codex 中配置使用该代理

### 方法1：通过 CC Switch UI 配置（推荐）

1. 打开 CC Switch 应用
2. 点击 "Providers" 或 "供应商" 标签
3. 点击 "Add Provider" 或 "添加供应商"
4. 填写配置：
   - **Name**: `MiMo 智能降级代理`
   - **Base URL**: `http://127.0.0.1:15722`
   - **API Key**: 你的 MiMo API Key（tp- 开头的 Token Plan Key）
   - **Model**: `mimo-v2.5-pro`（代理会自动处理降级）
5. 保存配置
6. 在 Codex 中选择这个新配置的供应商

### 方法2：直接修改 Codex 配置文件

如果你使用 Codex CLI，可以编辑配置文件：

```bash
# 找到 Codex 配置文件
# 通常在 ~/.codex/config.json 或类似位置

# 添加或修改 provider 配置
{
  "providers": [
    {
      "name": "MiMo 智能降级代理",
      "baseURL": "http://127.0.0.1:15722",
      "apiKey": "tp-coh626rf9chsglqvak5mc6ezz9k00n7rtosj78ehk30rc5dp",
      "model": "mimo-v2.5-pro"
    }
  ]
}
```

## 测试图片识别

配置完成后，在 Codex 中尝试：

1. **纯文本测试**：
   ```
   你好，请用中文介绍一下自己
   ```
   → 应该使用 mimo-v2.5-pro 响应

2. **图片测试**：
   - 发送一张图片给 Codex
   - 或者使用图片 URL
   - 问："这张图片是什么内容？"
   → 应该自动降级到 mimo-v2.5，识别图片内容

## 代理工作原理

```
用户请求 (Codex)
    ↓
本地代理 (127.0.0.1:15722)
    ↓ 检测是否包含图片
    ├─ 纯文本 → 使用 mimo-v2.5-pro
    └─ 含图片 → 自动切换到 mimo-v2.5
    ↓
转发到 MiMo API
    ↓
返回响应给 Codex
```

## 日志查看

代理运行时会在控制台输出日志：
- `[降级] 检测到图片内容，模型切换: mimo-v2.5-pro → mimo-v2.5`
- `[正常] 使用原模型: mimo-v2.5-pro（无图片内容）`

## 停止代理

在运行代理的终端按 `Ctrl+C` 或关闭终端窗口。

## 故障排查

1. **连接失败**：确认代理正在运行（端口 15722）
2. **API 错误**：检查 API Key 是否正确
3. **图片不识别**：查看代理日志确认是否触发降级
