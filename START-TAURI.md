# 启动 CC Switch Tauri 应用

## 快速启动命令

在终端中执行以下命令：

```bash
cd ~/Desktop/cc-switch
pnpm tauri dev
```

## 预期行为

1. **首次编译**：如果是第一次运行，Rust 编译可能需要 **5-10 分钟**
   - 你会看到 "Compiling..." 的输出
   - 编译完成后会自动打开应用窗口

2. **后续启动**：编译完成后，再次启动只需要 **10-30 秒**

3. **应用窗口**：编译完成后，CC Switch 应用窗口会自动打开

## 如何测试环境诊断功能

应用启动后：

1. 点击顶部的 **"设置"** 按钮（齿轮图标）
2. 点击 **"关于"** 标签页
3. 向下滚动，找到 **"本地环境检查"** 区域
4. 你会看到：
   - 环境状态卡片
   - 工具安装状态（Claude、Codex、Gemini、OpenCode）
   - "刷新" 按钮
   - "一键安装" 或 "一键修复" 按钮（根据诊断结果显示）

## 测试功能

### 1. 查看诊断结果
- 应该自动显示当前环境状态
- 显示哪些工具已安装/未安装
- 显示是否有环境问题

### 2. 测试刷新功能
- 点击 "刷新" 按钮
- 应该重新诊断环境并更新显示

### 3. 测试一键安装（可选）
**注意**：这会实际安装工具到你的系统
- 如果有工具未安装，会显示 "一键安装" 按钮
- 点击后会自动安装缺失的工具

### 4. 测试一键修复（可选）
**注意**：这会修改你的环境变量和配置文件
- 如果有环境冲突，会显示 "一键修复" 按钮
- 点击后会自动修复检测到的问题

### 5. 测试国际化
- 切换语言（中文/英文）
- 验证所有文案正确切换

## 如果遇到问题

### 编译卡住
如果 `pnpm tauri dev` 运行后没有任何输出：
1. 等待 2-3 分钟，Rust 编译器可能在后台工作
2. 如果超过 5 分钟还没有输出，按 Ctrl+C 停止，然后重试

### 端口占用
如果提示端口被占用：
```bash
# 查找占用端口的进程
lsof -i :3000
# 杀掉进程
kill -9 <PID>
```

### 清理重新编译
如果遇到编译错误：
```bash
cd ~/Desktop/cc-switch
rm -rf src-tauri/target
pnpm tauri dev
```
