# 2.7 Codex 记忆/Chronicle 功能

> 适用版本：CC Switch v3.16.5+（参考 [OpenAI Codex 记忆文档](https://developers.openai.com/codex/memories)）

Codex CLI 内置了「记忆」与「Chronicle」功能，会在每个会话结束后自动抽取要点、跨会话整合。`~/.codex/config.toml` 中通过 `[memories]` 段控制：

```toml
[memories]
generate_memories = true                       # bool：是否启用记忆生成
use_memories = true                            # bool：是否在新会话中注入已有记忆
disable_on_external_context = false            # bool：外部上下文（MCP / Web 搜索）触发的会话是否参与记忆生成
min_rate_limit_remaining_percent = 25          # int：剩余速率限制百分比阈值
extract_model = "MiniMax-M3"             # 用于逐线程记忆抽取的模型
consolidation_model = "gpt-5.4-mini"      # 用于跨线程记忆整合的模型
```

`extract_model` 与 `consolidation_model` 显式配置时，Codex 后端会调用这两个模型执行记忆处理。**若未配置，则回退到顶层 `model`**（即你当前供应商的主模型）。

## 常见问题：自定义供应商下记忆功能报错

### 现象

切到第三方/自定义供应商后，Codex 启动时报类似错误：

```
Codex memory/chronicle backend failed: model "MiniMax-M3" not found
```

或 `extract_model` / `consolidation_model` 报「模型不存在」。

### 原因

`[memories]` 段里硬编码的 `extract_model` / `consolidation_model` 指向的是某个**特定模型**（例如 `MiniMax-M3`、`gpt-5.4-mini`）。当你切换到一个**不提供该模型**的自定义供应商时，Codex 后端会因找不到模型而完全跳过记忆抽取/整合，记忆功能失效。

### 解决：使用「启用 Codex 记忆功能」开关

CC Switch v3.16.5+ 在 Codex 配置面板的 **config.toml (TOML) 编辑器**行新增了一个「**启用 Codex 记忆功能**」复选框，位于「启用远程压缩」和「写入通用配置」之间。

**勾选后**：

- cc-switch 会把当前供应商的 `model`（在「模型映射 / 模型目录」中维护）写入 `[memories].extract_model` 和 `[memories].consolidation_model`
- 切换到不同供应商后，下一次保存会自动同步新供应商的 model
- 其他 `[memories]` 字段（`generate_memories` / `use_memories` / `disable_on_external_context` / `min_rate_limit_remaining_percent`）保持不变

**取消勾选后**：

- 整个 `[memories]` 段被移除
- Codex 走默认回退路径（用顶层 `model`）

### 操作步骤

1. 打开要编辑的 Codex 供应商卡片，点击「编辑」
2. 在 `config.toml (TOML)` 编辑器一行的右侧找到「**启用 Codex 记忆功能**」复选框
3. 勾选该复选框
4. 点击「保存」
5. 重启 Codex CLI（按 Code 官方要求刷新模型列表与记忆配置）

### 适用场景

| 场景 | 是否需要勾选 |
|------|--------------|
| 使用 OpenAI 官方供应商 | 可选（Codex 默认回退到 `model`，显式勾选更稳定） |
| 使用第三方网关（DeepSeek / GLM / Qwen / 自建 OpenAI 兼容） | **推荐勾选**，避免模型名不匹配 |
| 「模型映射」配置了多个不同名字的模型 | **推荐勾选**，记忆模型跟随主模型自动切换 |
| 想要记忆用更便宜的模型而非主模型 | **不勾选**，手动在 `config.toml` 里写固定模型名即可 |

### 常见误解

- **勾选后无法填入自定义的 extract_model？** 这是设计上的简化：让记忆用与聊天同一模型，避免跨供应商不可用。若确实需要混合配置，可直接编辑 `config.toml` 后再取消勾选（cc-switch 不会覆盖你手动改的值，只要顶层没 `model` 字段）。
- **修改后 Codex 没生效？** 重启 Codex CLI——Codex 启动时才会读 `config.toml`，热更新不生效。
- **「记忆」和「Chronicle」是一回事吗？** Chronicle 是 Codex 的「跨会话叙事」功能，复用同一个 `[memories].consolidation_model` 字段。
