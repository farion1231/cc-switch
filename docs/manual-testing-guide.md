# CC Doctor 环境诊断与修复功能手动测试指引

## 测试环境
- 平台：macOS
- 应用：CC Doctor (Tauri 桌面应用)
- 分支：feature/environment-doctor
- 测试日期：2026年5月5日

## 启动应用

应用已通过以下命令启动：
```bash
cd ~/Desktop/cc-doctor
pnpm tauri dev
```

应用窗口应该会自动打开。如果没有，请检查终端输出是否有错误信息。

- 应该看到"环境状态"卡片

**检查点**：

### 第三步：测试一键安装功能（可选）

**注意**：此测试需要先卸载某个工具。建议跳过，除非你愿意卸载并重新安装。

**预期结果**：
- [ ] 按钮显示"安装中..."加载状态

### 第四步：测试一键修复功能（可选）

**注意**：此测试需要创建环境变量冲突。建议跳过，除非你愿意修改环境变量。

如果要测试：
```bash
# 创建环境变量冲突
export ANTHROPIC_API_KEY="test_key_conflict"
echo 'export ANTHROPIC_API_KEY="test_key_conflict"' >> ~/.zshrc
```

然后在应用中：
1. 重启应用或重新诊断
2. 应该看到"环境变量冲突：ANTHROPIC_API_KEY"问题
3. 点击"一键修复"按钮

**预期结果**：
- [ ] 按钮显示"修复中..."加载状态
- [ ] 自动备份并删除冲突的环境变量
- [ ] 修复成功后显示 toast 提示："成功修复 1 个问题"
- [ ] 自动重新诊断，冲突消失

**验证修复**：
```bash
# 检查环境变量是否已删除
echo $ANTHROPIC_API_KEY  # 应该为空
grep ANTHROPIC_API_KEY ~/.zshrc  # 应该找不到
```

**恢复**：
```bash
# 清理测试数据
unset ANTHROPIC_API_KEY
sed -i '' '/ANTHROPIC_API_KEY/d' ~/.zshrc
```
